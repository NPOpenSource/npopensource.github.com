环境搭建

我建议按照facebook 官方文档进行安装。

1 安装 Homebrew

打开终端

复制下面代码到命令行
ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

2 安装 nvm

复制下面代码到命令行
curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.26.1/install.sh | bash

要是每次都要你重新开启nvm 
进行下面操作
cd ~

 open ~/.bashrc 或者 open ~/.profile 或者 open ~/.zshrc 三个命令中的一个执行

这时候会打开上面三个文件中的一个
将下面命令复制到文件最后即可。

. ~/.nvm/nvm.sh

3 安装 node.js

可以用命令行。也可以打开官网下载好nodejs 安装
这里先说明下，nodejs必须是4.0版本或者更好版本。

我遇到的一个问题。由于我以前安装过node.js 这里又安装一个。导致我安装的react-native 不好使。 我的解决办法是把安装过的nodejs全删掉。特别是 用户目录下 隐藏目录 .rvm 目录中的 nodejs

命令行安装 nvm install node && nvm alias default node
挺慢的

4 安装 watchman

命令行 brew install watchman

5 安装 flow

命令行 brew install flow

react-native 开发工具

NO1: sublime text 3

我们在开发 react-native 之前需要为sublime加载几个插件。

1、package controller

操作：使用快捷键 ctrl+` 或者是 View>Show Console 控制台，把下面的内容复制后回车，安装成功后重新启动sublime

接下来我们按 快捷键 shift + commad + p 打开package controller

输入package controller:install package 回车，然后输入我们要的插件就可以install下来了。非常方便。

2、React-native-snippets

这个插件快速搭建一个RN开发环境还有一些生命周期的语法。(具体操作请看https://packagecontrol.io/packages/react-native-snippets)

3、sublime-react

RN是基于React.js的，都是用JSX语法，sublime-react是用来显示语法颜色（你懂得，代码看起来漂亮）

4、babel

babel插件支持ES6语法和JSX语法，要比sublime-react添加更多关键字，看起来更舒服。出现问题也会提示。

到这里react-native的开发工具环境就搭建完成啦！

怎么用sublime text3 的自动提示 按住shift +commit+p 选择自己要的函数即可

NO2:Atom

6月15日Atom1.0正式发布，3月份Facebook宣布为React和React-native开发并开源一款基于Atom的IDE–unclide ,很高兴，6月22日在nuclide.io 测试版发布。相信过不了多久，一款强大的RN编辑器就出来了。(RN狗福音!) （进过测试，Atom在install nuclide之后会变得很卡，目前不建议各位尝试!请耐心等待）

Atom的操作流程跟sublime基本差不多。这里就不做说明！

下面将AppDelete.m 中的localhost

jsCodeLocation = [NSURL URLWithString:@"http://localhost:8081/index.ios.bundle"];

react几个函数的简单学习

（本来打算直接写控件的，但是像我这样从来没有接触过react的人看api实在是看不懂，没办法，只能硬着头皮学点react的皮毛了）

React.createClass() 相关的函数和生命周期

 1.ReactElement render()
必须实现的函数   

当call render的时候， 他会检测 this.props 和  this.state 返回一个 a single child element


2.object getInitialState()   
部件没有渲染的时候调用这个函数。调用在 render之前，返回的object 作为 this.state 的值

3.object getDefaultProps()
类创建的时候，将object的值赋值给this.props

4.object propTypes
5.array mixins

6.object statics 
定义类的静态方法

7.string displayName 
用来debug的

下面是生命周期函数

8.componentWillMount()
render 渲染之前 ，服务器和客户端只调用一次。你可以 在这个函数 setState ,render 只检测 state 一次。

9.componentDidMount()
当render 之后 只有客户端调用一次 该函数

10.componentWillReceiveProps(
    object nextProps
)
当component 有新的props 该方法调用

11.boolean shouldComponentUpdate(
    object nextProps, object nextState
)

当 render 之前  新的props 和 state 有了之后调用 这个函数不能用来初始化render 和 强制更新 返回的结果是false 就不更新了

12.componentWillUpdate(
    object nextProps, object nextState
)

当新的props 和state 设置了，而render之前，这个函数被调用。不能用来init render 

13.componentDidUpdate(
    object prevProps, object prevState
)
当component 刷新了这个立即调用

14.componentWillUnmount()
当component 被卸载之前被调用

以上就是大概的react 学习。 其实还应该学习react 的语法的，但是算了吧。比较简单。模仿别人写的工程直接干了。
